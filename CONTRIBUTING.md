# Contributing to Website Health Monitoring System

First off, thank you for considering contributing to the Website Health Monitoring System! It's people like you that make the open-source community such a fantastic place to learn, inspire, and create.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) first. If it doesn't exist yet, feel free to open a new one.

## 2. Fork & create a branch

If this is something you think you can fix, then fork the repository and create a branch with a descriptive name.

```bash
git checkout -b feature/your-feature-name
```

## 3. Implementation Guidelines

- Ensure your Bash script is compatible with `bash v4.0+`.
- Run tests before submitting.
- Use `set -euo pipefail` for strict error handling.
- Keep output formats JSON and CSV compliant.
- Document any new features in the `README.md` and `CHANGELOG.md`.

## 4. Make a Pull Request

Once you're finished with your commits, create a pull request (PR).
- Fill out the PR template.
- Ensure your PR describes the problem it solves.

Thank you!
